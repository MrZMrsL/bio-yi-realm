# 生化易界 · 游戏设计增强方案 v1.0

> 基于当前 v8.2 版本的分析与设计建议
> 2026-06-05 | GameDesigner

---

## 一、现状诊断

### 关键发现

| 维度 | 现状 | 问题等级 |
|------|------|:--------:|
| **战斗公式** | 伤害 = (atk - def) × 1.5 × 元素 × 连击 | ⚠️ |
| **连击机制** | 每连击+0.5倍，3连重置，答错归零 | ✅ |
| **敌人成长** | 每层HP+8%、ATK+7%、DEF+5% | ⚠️ |
| **XP曲线** | 每级需求 = 100 × 1.15^(level-1) | ⚠️ |
| **怪物收养** | 80/80 敌人全部可收养，无稀有度 | 🔴 |
| **宠物系统** | 提供10%属性加成，无养成深度 | 🔴 |
| **经济系统** | 金币来源多、消耗少，无通胀控制 | 🔴 |
| **装备** | 武器/防具/饰品三槽，掉落随机 | ⚠️ |
| **专精差异** | 仅影响出题范围，战斗差异有限 | ⚠️ |

### 🔴 核心问题：怪物收养没有"稀有感"

100%的敌人都可收养、且收养条件完全一样——这让收养系统失去了"抓宠"的核心乐趣。玩家没有目标感，没有"这只我一定要抓到！"的冲动。

---

## 二、设计改动提案

### 改动 A：怪物稀有度分层（高优先级）

**目标**：让收养有目标感、有成就感

**设计**：

```
稀有度      占比      颜色      额外条件
─────────────────────────────────────────────
普通 Common   60%     灰色      答对2题即可收养
稀有 Rare     25%     蓝色      答对3题 + 答错不超过1次
史诗 Epic     12%     紫色      答对3题 + 全部答对
传说 Legendary 3%     金色      答对3题全对 + 特殊Boss战
```

**实现方式**：

- 敌人数据新增 `rarity: 'common' | 'rare' | 'epic' | 'legendary'` 字段
- 高稀有度敌人提供 **更高的宠物属性加成**（史诗/传说级的宠物攻击加成翻倍）
- 高稀有度敌人的 **初始属性更高**（同楼层下更强）
- 传说级敌人需要 **先打败它，再通过收养考验**，再打一场Boss战

**为什么这种分层有效**：
- 给玩家一个"刷"的目标——"我想要那个紫色稀有度的怪物"
- 答对3题全对才有史诗——技术上有挑战性
- 传说敌人需要多打一场Boss战——投入感更强
- 不同稀有度的宠物加成不同——有实际收益驱动

---

### 改动 B：宠物融合进化系统（中优先级）

**目标**：让抓到的宠物有养成深度

**设计**：

```
两只同元素/同稀有度的宠物 → 融合 → 更高级的进化形态
   ↓
进化后：外观升级、属性翻倍、可能解锁特殊能力
```

| 融合门槛 | 结果 | 属性提升 |
|----------|------|:--------:|
| 2只普通 → | 稀有级 | +50% 基础属性 |
| 2只稀有 → | 史诗级 | +100% 基础属性 |
| 2只史诗 → | 传说级 | +200% 基础属性 + 特殊技能 |

**经济关联**：每次融合消耗金币，稀有度越高消耗越大。创造金币消耗场景，解决通胀问题。

---

### 改动 C：伤害公式重构（中优先级）

**目标**：让战斗更有策略性，避免"数值碾压"或"刮痧"

**当前公式问题**：

```
当前：damage = (atk - def) × 1.5 × element × combo
```

问题1：如果 `atk < def`，你只会打1点伤害——哪怕连击再高也没用。
问题2：`1.5` 的基数是硬编码的，玩家不知道为什么是1.5。
问题3：敌人有 `minDmg = max(3, atk × 0.4)` 保底，玩家没有。

**建议新公式**：

```
玩家伤害 = atk × (1 + subjectBonus) × elementMultiplier × comboMultiplier - def × 0.5
          （最低保底 atk × 0.3）

敌人伤害 = enemy.atk × elementMultiplier - def × 0.5
          （最低保底 enemy.atk × 0.4）
```

**改动效果**：
- 防御不再是"减法"而是"减半"——前期不刮痧，后期不无敌
- 玩家也有保底伤害——答对永远有回报
- `subjectBonus` 根据当前题目学科和专精匹配度加成——让专精选择更有意义

---

### 改动 D：经济系统重构

**目标**：建立健康的金币循环，避免通货膨胀

**当前问题**：
- 金币只有商店和锻造两个消耗场景
- 商店物品买完就不需要了
- 没有持续消耗

**建议新增消耗场景**：

| 消耗场景 | 费用 | 备注 |
|---------|:----:|------|
| 宠物融合手续费 | 500~5000g | 稀有度越高越贵 |
| 重置属性点（已有） | level × 50g | ✅ 已实现 |
| 装备强化 | 100~5000g | 消耗金币升级装备等级 |
| 钓鱼次数刷新 | 200g | 不高兴等刷新？花钱 |
| 答题"跳过" | 500g | 遇到不会的题可以付费跳过 |

---

### 改动 E：连击/学科深度

**目标**：让三大学科在战斗中真的有不同手感

| 学科 | 连击特性 | 视觉效果 | 专精优势 |
|------|---------|----------|----------|
| 🧪 化学 | 伤害递增更快（×0.7/连） | 火焰/爆炸粒子 | 攻高防低，玻璃大炮 |
| 🧬 生物 | 答对回血（5%×连击） | 绿色治疗粒子 | 持久战，血量优势 |
| ☯️ 易学 | 连击不重置（答错保留一半） | 紫色卦象粒子 | 容错率高，后期发力 |

**实现方式**（极小改动）：
- 在 `answerAttack` 中根据 `question.subject` 应用不同连击加成
- 化学：`comboMultiplier = 1 + consecutiveCorrect * 0.7`
- 生物：每次答对额外回复 `maxHp × 0.02 × consecutiveCorrect`
- 易学：答错时 `consecutiveCorrect = Math.floor(consecutiveCorrect / 2)` 而非归零

---

### 改动 F：限时Boss优化

**当前**：限时Boss只是倒计时打伤害，赢了给奖励。
**建议**：限时Boss改为"每周挑战"形式：

- 每周一个主题学科（化学周/生物周/易学周轮流）
- Boss有三个阶段，每个阶段切换一次行为模式
- 击杀Boss后可获得 **该周专属宠物蛋**（孵化后获得稀有宠物）
- 每周排名根据 **击杀时间** 或 **伤害量** 排序

### 当前数值模拟（v8.3 新公式）

以Lv.5化学专精玩家挑战5层敌人为例：

```
当前玩家属性：atk=22, def=8, hp=170
当前5层敌人：hp≈80, atk≈19, def≈6

新公式：damage = max(atk×0.3, atk×m - def×0.5)
敌人公式：damage = max(atk×0.3, atk×m - def×0.5)

┌──────────────────────────────────────┐
│ 第一轮答对（无连击，元素平克）        │
│ damage = max(22×0.3=6, 22×1-6×0.5=19)│  ← 至少6点保底
│ = 19  → 80/19 ≈ 4.2下击杀           │
├──────────────────────────────────────┤
│ 两连击（化学）                       │
│ damage = max(6, 22×1.7-6×0.5=34.4)  │  ← 34伤害，刮痧不再
│ = 34  → 80/34 ≈ 2.3下               │
├──────────────────────────────────────┤
│ 三连暴击（化学，元素克制）           │
│ damage = max(6, 22×3.1×1.5-3=99.3)  │  ← 秒杀！暴击感拉满
│ = 99  → 秒杀！                       │
└──────────────────────────────────────┘
```
