# 生化易界 游戏开发质检清单（QA Checklist）

> **红线原则：不质检不push。质检未通过，禁止推送至 origin/master。**
> 
> 违反后果：可能导致线上版本崩溃、数据丢失、玩家体验受损。

---

## 一、构建检查（Build Verification）

### 1.1 构建零报错
- [ ] `npm run build` 或 `vite build` 零错误（error = 0）
- [ ] 零警告（warnings 需逐条确认非致命）
- [ ] 控制台无未定义变量、未使用 import、类型错误

### 1.2 构建产物检查
- [ ] `dist/` 目录生成成功
- [ ] 文件大小无明显异常膨胀（单个JS > 500KB 需检查）
- [ ] 静态资源（图片、字体）正确打包

---

## 二、功能验证（Functional QA）

### 2.1 核心流程验证
- [ ] 游戏启动 → 正常进入主界面
- [ ] 地牢探索 → 进入准备 → 房间选择 → 战斗 → 胜利/失败 → 返回
- [ ] 战斗系统 → 答题攻击 → 连击 → 暴击特效 → 胜利/捕捉
- [ ] 钓鱼系统 → 抛竿 → 咬钩 → 收竿 → 普通鱼/古籍/传说鱼处理
- [ ] 怪物农场 → 捕捉 → 升级 → 放生 → 跟随
- [ ] 图鉴系统 → 新发现通知 → 收集进度
- [ ] 错题本 → 记录 → 复习 → 标记掌握
- [ ] 存档系统 → 保存 → 加载 → 数据完整性

### 2.2 数据持久化验证
- [ ] 战斗后退出地牢 → 重新进入 → 数据不丢失
- [ ] 钓鱼后返回主界面 → 再进入钓鱼 → 次数/收藏保留
- [ ] 浏览器刷新 → 自动加载存档 → 数据完整
- [ ] 装备/材料/金币/经验 → 各字段正确保存

### 2.3 新增功能专项验证
每次新增功能后，必须逐项验证：
- [ ] 新功能正常触发（触发条件、触发概率）
- [ ] 新功能边界条件（最大值、最小值、空值）
- [ ] 新功能异常处理（网络中断、数据异常、用户取消）
- [ ] 新功能与现有系统冲突（数据覆盖、状态混乱）

---

## 三、代码质量检查（Code Quality）

### 3.1 语法与逻辑
- [ ] 无未定义变量引用（特别是 `store.xxx` 未在 return 中导出）
- [ ] 无死代码（注释掉的代码、未使用的函数）
- [ ] 循环引用检查（A导入B，B导入A）
- [ ] 异步操作有错误处理（try/catch、.catch()）

### 3.2 数据一致性
- [ ] 新字段加入存档结构 → 旧存档兼容（默认值处理）
- [ ] 新状态变量 → 在 `saveGame()` 和 `loadGame()` 中同步
- [ ] 计算属性（computed）依赖项完整
- [ ] 响应式数据修改通过正确方式（Vue3 ref/value）

### 3.3 性能检查
- [ ] 无内存泄漏（setInterval/setTimeout 有清理）
- [ ] 大数据循环有优化（防抖、节流、虚拟列表）
- [ ] 图片/音频资源懒加载

---

## 四、兼容性检查（Compatibility）

- [ ] Chrome 最新版正常
- [ ] Edge 最新版正常
- [ ] Firefox 最新版正常（如有条件）
- [ ] 移动端浏览器基本可用（布局不崩、按钮可点击）
- [ ] 不同屏幕尺寸（1920x1080、1366x768、手机竖屏）

---

## 五、用户体验检查（UX）

- [ ] 按钮可点击区域足够大（最小 44x44px）
- [ ] 文字对比度足够（浅色文字在深色背景上清晰）
- [ ] 加载状态有提示（不要让用户以为卡死）
- [ ] 错误操作有反馈（按钮不可点时 visual 反馈）
- [ ] 动画不卡顿、不闪屏
- [ ] 音效开关有效、不突兀

---

## 六、安全与数据

- [ ] 无敏感信息泄露（API密钥、密码等）
- [ ] localStorage 数据有版本控制（存档结构变更时升级）
- [ ] 用户输入有基本过滤（防止 XSS）

---

## 七、质检流程

```
开发完成 → 自检（本清单） → 浏览器实际测试 → 全部通过 → git commit → git push
                                    ↓
                              任一项目不通过 → 修复 → 重新质检
```
